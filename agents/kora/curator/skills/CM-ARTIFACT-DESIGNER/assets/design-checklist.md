# Design Checklist

- Tipo correcto: `KORA/MD` o `KORA/Spec-MD`
- URN conceptual sin version embebida
- Namespace y subdirectorio destino definidos
- Fuente y `provenance.source` previsibles
- Estrategia `directa|segmentada|segmentada+fidelidad-reforzada`
- Familia documental determinada cuando aplique
- Frontmatter inicial viable para `drafts/`
- Headings primarios recuperables desde el diseno
