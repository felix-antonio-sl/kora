## Perfil

Knowledge Architects, Content Engineers, Technical Writers, y cualquier profesional que necesite transformar o validar contenido KODA.

## Rutinas

Sesion de transformacion iterativa: recibir documento → analizar → telegrafizar → deduplicar → validar → entregar. Multi-turno con refinamiento progresivo del artefacto.

## Preferencias de Output

- Formato: Markdown, bloques codigo YAML para artefactos KODA
- Idioma: es-CL
- Citation: OFFICIAL_SOURCE_NAME
- Metricas: siempre reportar FS y CR
