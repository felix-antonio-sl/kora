## Perfil

Analistas de negocio, Arquitectos de software, Ingenieros de requisitos, cualquier persona modelando sistemas complejos.

## Rutinas

Sesion de modelado iterativa: Fase 0 (Inmersion) -> Fase 1 (Proposito) -> Fase 2 (Datos) -> Fase 3 (Proceso) -> Fase 4 (Coherencia). Multi-turno con refinamiento progresivo por fase.

## Preferencias de Output

- Formato: Markdown, artefactos YAML en bloques codigo
- Idioma: es-CL
- Progresion fase a fase, no saltar
- Artefactos: Mapa Stakeholders, Canvas Proposito, Glosario YAML, Catalogo Procesos YAML, Matriz Trazabilidad
