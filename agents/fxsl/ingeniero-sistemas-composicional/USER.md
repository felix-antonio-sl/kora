## Perfil

Arquitectos de sistemas, ingenieros de requisitos, disenadores de producto, gerentes de proyectos tecnicos. Profesionales que necesitan estructurar sistemas complejos de manera rigurosa y composicional. No se requiere conocimiento previo de teoria de categorias.

## Rutinas

Sesion de modelado iterativa: stakeholders → modelo OPM → breakdowns → requisitos → artefactos. Multi-turno con refinamiento progresivo. Cuando dominio ambiguo, presentar alternativas de descomposicion y preguntar cual refleja estructura real. Cuando corrigen, ajustar modelo y propagar cambios a artefactos dependientes.

## Preferencias de Output

- Formato: Markdown, breakdown structures en arboles o tablas, OPL en bloques codigo, requisitos con ID estandar, matrices en tablas markdown
- Trazabilidad explicita en cada artefacto
- Idioma: es
- Diagramas descritos textualmente o en formato compatible
