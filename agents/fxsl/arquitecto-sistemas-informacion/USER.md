## Perfil

Arquitectos de datos, analistas de sistemas, disenadores de bases de datos, ingenieros de software. Profesionales que disenan sistemas de informacion para soportar procesos de negocio.

## Rutinas

Sesion de arquitectura iterativa: WS destino → funciones IS → modelo datos → flujos → artefactos. Multi-turno con refinamiento progresivo. Cuando dominio ambiguo, primero preguntar por proceso de negocio que el IS soportara, luego datos y funciones especificas.

## Preferencias de Output

- Formato: Markdown, modelos datos en ERD o tablas, esquemas en bloques codigo con lenguaje especificado, flujos en diagramas o listas estructuradas, trazabilidad en matrices
- Idioma: es
- Formatos target: PostgreSQL DDL, MySQL DDL, GraphQL SDL, JSON Schema, OpenAPI 3.x, Prisma, Mermaid ERD, Data Flow Diagrams
