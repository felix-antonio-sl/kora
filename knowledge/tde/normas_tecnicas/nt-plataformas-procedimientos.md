# DS 11 — Norma Tecnica de Calidad y Funcionamiento de Plataformas Electronicas

- **Promulgacion**: 19-MAY-2023
- **Publicacion**: 17-AGO-2023
- **Base legal**: Ley 21.180 (TDE) + DS 4/2020 (Reglamento) + DFL 1/2020 (Gradualidad)
- **Revision**: cada 2 anios desde vigencia

## Objeto (art. 1)

Forma en que organos AE deben: mitigar obsolescencia tecnologica, mantener continuidad operacional, establecer/medir niveles servicio, aumentar resiliencia, brindar soporte, informar TyC uso, monitorear funcionamiento de plataformas electronicas relacionadas con procedimientos administrativos.

## Definiciones (art. 2)

| Termino | Definicion |
|---------|-----------|
| Ciclo Gestion Calidad | Proceso mejora continua plataformas → reduccion progresiva brechas tecnologicas |
| Diagnostico | Evaluacion periodica dentro Ciclo Gestion Calidad |
| Caracteristicas/Subcaracteristicas | Categorias que agrupan atributos para evaluar calidad plataformas |
| EvalTIC | Evaluacion Proyectos TIC (DIPRES + ChileCompra + DGD); recomendacion tecnica formulacion presupuestaria |
| Metrica | Medida de caracteristica/subcaracteristica en momento del tiempo |
| Plan de Mejora Continua | Diagnostico + iniciativas mejoras + responsables para periodo determinado |
| Plataforma electronica | Software + datos + infraestructura que sustenta procesos/procedimientos |

## Etapas previas al Ciclo

### Catalogo de plataformas (art. 3)
- Cada organo elabora catalogo plataformas bajo administracion
- Linea de base: estado inicial plataforma al ingresar al Ciclo
- Actualizacion segun art. 11

## Ciclo de Gestion de la Calidad (arts. 4-10)

### Alcance (art. 4)
Mejora continua materializada en diseno + implementacion Plan de Mejora Continua. Cada organo prioriza plataformas segun competencias, objetivos estrategicos, disponibilidad presupuestaria, fases Ley 21.180.

### 5 Etapas (art. 5)

| Etapa | Nombre | Contenido |
|-------|--------|-----------|
| 1 | Elaboracion Plan | Identificacion/medicion estandares, diagnostico, iniciativas mejora, objetivos |
| 2 | Formulacion presupuestaria | Solicitudes presupuesto; asociar EvalTIC cuando corresponda |
| 3 | Implementacion mejoras | Ejecutar iniciativas segun Plan + presupuesto asignado |
| 4 | Evaluacion calidad | Evaluar efectos iniciativas vs estandares; reduccion brechas; cumplimiento objetivos |
| 5 | Ajustes y consolidacion | Correcciones post-evaluacion; actualizacion catalogo plataformas |

### Elaboracion Plan detalle (art. 6)

8 pasos:
1. **Identificar plataformas** a mejorar (priorizacion art. 4)
2. **Identificar/medir estandares** de otras NTs (art. 57 Reglamento)
3. **Describir iniciativas** mejora: objetivos, areas responsables, plazos
4. **Seleccionar subcaracteristicas**: minimo 4 (de 1 o varias caracteristicas)
5. **Definir metricas y umbrales**: exitosa / aceptable / insuficiente
6. **Determinar costos y plazos**: estimacion + hoja de ruta con hitos
7. **Estimar vida util** mejoras (principio actualizacion art. 16 bis Ley 19.880)
8. **Identificar riesgos** asociados al Plan

Considerar Guia Lineamientos Desarrollo Software (DGD). Plan remitido a SEGPRES a solicitud (art. 55 Reglamento).

### Formulacion presupuestaria (art. 7)
- Solicitar recursos a DIPRES o formulacion interna
- Asociar EvalTIC cuando corresponda
- Ajustar Plan segun recomendaciones/disponibilidad

### Implementacion (art. 8)
Cada organo define gobernanza interna + mecanismos aseguramiento.

### Evaluacion calidad (art. 9)
- a) **Diseno evaluacion**: actividades, momentos, recursos/herramientas
- b) **Ejecucion**: comparar resultados vs metricas/umbrales Etapa 1
- c) **Informe evaluacion**: contenido minimo:
  - i) Identificacion Plan
  - ii) Por cada mejora: descripcion, fechas, resultado mediciones, mediciones anteriores, analisis
  - iii) Conclusiones
  - iv) Recomendaciones proximo Plan

### Ajustes y consolidacion (art. 10)
- Resultado insuficiente → ajustes obligatorios para alcanzar objetivos
- Resultado aceptable → organo decide si ajusta
- Completada etapa cierre → inicio siguiente Plan

### Actualizacion catalogo (art. 11)
Post Plan: actualizar catalogo con descripcion mejoras implementadas.

## EvalTIC (arts. 12-14)

### Formulacion proyectos (art. 12)
- Organos formulan proyectos TIC segun disposiciones Ley Presupuestos/instrucciones
- Disposiciones publicadas en web DGD

### Plataforma (art. 13)
- DIPRES dispone plataforma via interfaz formulacion presupuestaria
- Administrador mantiene actualizada (cambios estado proyectos)

### Evaluacion (art. 14)
- DIPRES + DGD + Sec. Modernizacion Hacienda + ChileCompra evaluan viabilidad tecnologica
- Evaluacion por si o evaluadores externos capacitados
- Calificacion segun Guia Formulacion Proyectos TIC periodo
- **Evaluaciones vinculantes** para formulaciones presupuestarias

## Coordinador de Transformacion Digital (arts. 15-16)

### Designacion (art. 15)
- Jefe Superior Servicio designa Coordinador TD
- Responsable: desarrollar/gestionar Plan Mejora Continua + iniciativas cumplimiento Ley 21.180
- Reporta directamente a Jefe Superior Servicio
- Contraparte DGD para avances implementacion + indicadores cumplimiento
- Comunicar designacion/cambio oficialmente a Jefe DGD
- DGD establece Red de Coordinadores TD

### Rol (art. 16)
- Informar internamente: plazos, indicadores, hitos, capacitaciones, temas TD
- Participar activamente en Red Coordinadores
- Reportar periodicamente avances TD y cumplimiento normativo (segun directrices DGD)

## Disposiciones finales

- **Guia tecnica** (art. 17): DGD dicta guias aspectos operativos
- **Gradualidad** (art. 18): segun DFL 1/2020 SEGPRES
- **Revision** (art. 19): cada 2 anios minimo
