# OPM — Glosario ISO 19450 (83 terminos)

Terminos oficiales ISO 19450. Para conceptos ampliados ver [→ 01-ontologia-opm.md].

---

## Terminos A–F

| Termino | Definicion |
|---------|-----------|
| Abstraction | Reduccion de detalle para mejorar comprension |
| Affectee | Transformee afectado por un proceso con cambio de estado. Solo objetos stateful |
| Agent | Habilitador humano o grupo humano |
| Attribute | Objeto que caracteriza algo distinto a si mismo |
| Behaviour | Transformacion de objetos via ejecucion de modelo OPM |
| Beneficiary | Stakeholder que obtiene valor funcional de la operacion del sistema |
| Class | Coleccion de things con misma esencia de perseverancia, valores de afiliacion, features y estados |
| Completeness | Grado en que todos los detalles del sistema estan especificados en el modelo |
| Condition Link | Enlace procedimental de objeto/estado-objeto a proceso imponiendo restriccion procedimental |
| Consumee | Transformee consumido o eliminado por un proceso |
| Context | Porcion de modelo OPM representada por un OPD y su OPL |
| Control Link | Enlace procedimental con semantica de control adicional |
| Control Modifier | Simbolo en enlace que agrega semantica de control: 'e' (evento) o 'c' (condicion) |
| Discriminating Attribute | Atributo cuyos valores distinguen relaciones de especializacion correspondientes |
| Effect | Cambio de estado en objeto o atributo. Solo aplica a objetos stateful |
| Element | Thing o enlace |
| Enabler | Entidad (humana o no humana) que habilita un proceso |
| Event | Punto en tiempo cuando objeto es creado, aparece, o transiciona a estado particular |
| Event Link | Control link representando evento originado en objeto/estado que habilita proceso |
| Exhibitor | Thing caracterizado por un feature |
| Feature | Atributo u operacion |
| Folding | Abstraccion ocultando refinamientos de refinee desplegado. Inverso de unfolding |
| Function | Proceso que provee valor funcional a beneficiario |

## Terminos G–O

| Termino | Definicion |
|---------|-----------|
| General | Refineable OPM con especializaciones |
| Informatical | Perteneciente a datos, informacion o conocimiento |
| Inheritance | Asignacion de elementos OPM desde general a sus especializaciones |
| Input Link | Enlace desde estado fuente de objeto al proceso transformador |
| Instance | Objeto o proceso en contexto de modelo u operacional |
| Instance (Operational) | Objeto o proceso unicamente identificable en runtime |
| Instrument | Habilitador no humano |
| Invocation | Iniciacion de proceso por otro proceso |
| Involved Object Set | Union de preprocess y postprocess object sets |
| In-Zoom Context | Things y enlaces dentro de limite de objeto in-zoomed |
| In-Zooming (Object) | Despliegue revelando objetos constituyentes y orden espacial |
| In-Zooming (Process) | Despliegue revelando procesos constituyentes y orden temporal |
| Link | Expresion grafica de relacion estructural o procedimental |
| Metamodel | Modelo de lenguaje de modelado |
| Model Fact | Relacion entre dos elementos OPM |
| Object | Elemento del modelo representando thing con existencia fisica o informatica potencial |
| Object Class | Patron para objetos con misma estructura y patron de transformacion |
| Object-Process Diagram (OPD) | Representacion grafica de modelo OPM |
| Object-Process Language (OPL) | Representacion textual de modelo OPM |
| Object-Process Methodology (OPM) | Lenguaje bimodal para modelado de sistemas complejos |
| OPD Object Tree | Grafo elaborando objeto via refinamiento |
| OPD Process Tree | Grafo elaborando procesos via in-zooming |
| Operation | Proceso que caracteriza un thing |
| Output Link | Enlace desde proceso transformador a estado de salida de objeto |
| Out-Zooming (Object) | Inverso de in-zooming de objeto |
| Out-Zooming (Process) | Inverso de in-zooming de proceso |

## Terminos P–Z

| Termino | Definicion |
|---------|-----------|
| Perseverance | Propiedad que define caracteristicas estaticas o dinamicas de things |
| Postcondition | Condicion resultante de completar proceso exitosamente |
| Postprocess Object Set | Objetos resultantes de completar proceso |
| Precondition | Condicion requerida para iniciar proceso |
| Preprocess Object Set | Objetos evaluados antes de iniciar proceso |
| Primary Essence | Esencia predominante (fisica o informatica) de un sistema |
| Procedural Link | Notacion grafica de relaciones procedurales |
| Procedural Relation | Conexion dependiente del tiempo o condicional entre objetos y procesos |
| Process | Transformacion de objetos en el sistema |
| Process Class | Patron para procesos con mismo patron de transformacion |
| Property | Anotaciones que distinguen elementos |
| Refineable | Elemento OPM susceptible de refinamiento |
| Refinee | Elemento elaborado por refinamiento |
| Refinement | Elaboracion que incrementa detalle y completitud del modelo |
| Resultee | Transformee creado por proceso |
| Stakeholder | Individuos o grupos con interes en el sistema |
| Stateful Object | Objeto con estados especificados |
| Stateless Object | Objeto sin estados especificados |
| State (Object) | Posible situacion o posicion de un objeto |
| State (System) | Snapshot del modelo del sistema en momento especifico |
| State Expression | Refinamiento revelando subconjuntos de estados de objeto |
| State Suppression | Abstraccion ocultando subconjuntos de estados de objeto |
| Structural Link | Notacion grafica de relaciones estructurales |
| Structural Relation | Conexiones invariantes entre elementos del sistema |
| Structure | Relaciones no transientes entre objetos en modelo OPM |
| System Diagram (SD) | OPD raiz que representa contexto de nivel superior del sistema |
| Thing | Objeto o proceso |
| Transformation | Creacion, consumo o cambio de estado de objeto |
| Transformee | Objeto afectado por proceso |
| Transforming Link | Enlace de consumo, efecto o resultado |
| Unfolding | Refinamiento agregando detalle a refinee |
| Value (Attribute) | Estado de atributo |
| Value (Functional) | Beneficio derivado de la funcion del sistema |
| Whole | Agregado |
